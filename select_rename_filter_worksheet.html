dplyr Cheat Sheet: Selecting Columns

Keeping, dropping, reordering, and combining variables with precision

SI ANALYTICS
PAGE 01 / 03
Purpose. Use dplyr::select() to keep, drop, and reorder columns in a data frame. Practise with a compact subset of the Yaoundé survey data.
Core idea. Selecting variables is often the first step in a clean, reproducible data workflow.

A Select by name or position

yao <- yaounde %>% 
  select(age, sex, highest_education,
         occupation, is_smoker, is_pregnant,
         igg_result, igm_result)

B Select ranges

yao %>% select(age:occupation)
yao %>% select(1:4)
5. “The colon operator selects a consecutive range of columns.”

C Drop columns with !

yao %>% select(!age)
yao %>% select(!age:occupation)
yao %>% select(!c(age, sex, igg_result))

D Pattern helpers

yao %>% select(starts_with("is_"))
yao %>% select(ends_with("_result"))
yaounde %>% select(contains("drug"))

E Reorder with everything()

yaounde %>% 
  select(is_pregnant, everything())

yaounde %>% 
  select(ends_with("result"), everything())
12. “everything() matches columns that have not yet been selected.”

dplyr Cheat Sheet: Renaming Columns

Creating clearer, analysis-ready variable names

SI ANALYTICS
PAGE 02 / 03
Purpose. Use rename() when you want to change names while retaining all columns. Use select() when you want to rename while keeping only selected columns.
Watch out. The syntax is new_name = old_name.

A Rename directly

yaounde %>% 
  rename(patient_age = age,
         patient_sex = sex)
15. “Inside rename(), the new name comes before the old name.”

B Rename while selecting

yaounde %>% 
  select(patient_age = age,
         patient_sex = sex)
18. “select() can rename columns while selecting them.”

C Choose the right verb

20. You want to keep every column but change sex to patient_sex.
21. You want a two-column table containing renamed versions of age and sex.

D Mini integration task

Create a compact table that places test results first, followed by all other variables, then renames sex to patient_sex.

Hint options: combine select(), everything(), and rename().

E Compact comparison

VerbUse it to…Example
select()Keep, drop, reorder, or rename selected columnsselect(patient_age = age)
rename()Change names while retaining other columnsrename(patient_age = age)
Read the syntax aloud: create patient_age from the existing age column.

dplyr Cheat Sheet: Filtering Rows

Keeping observations that meet defined conditions

SI ANALYTICS
PAGE 03 / 03
Purpose. Use filter() to retain rows that satisfy clear conditions. Combine relational and logical operators to define meaningful subsets.
Key distinction. select() changes columns. filter() changes rows.

A Filter with one condition

yao %>% filter(sex == "Male")
yao %>% filter(age < 6)
yao %>% filter(age >= 70)
24. “== tests equality, while = is often used to specify function arguments.”
Hint options: use filter() and ==.
Hint options: use age < 18.

B Relational and membership operators

OperatorMeaningExample
<Less thanage < 18

!=Not equal tosex != "Male"
%in%Appears in a setneighborhood %in% c("Tsinga","Messa")
Hint options: put both neighbourhood names inside c().

C Combine conditions

yao %>% 
  filter(is_pregnant == "Yes" & 
         is_smoker == "Ex-smoker")

yao %>% 
  filter(is_pregnant == "Yes" | 
         is_smoker == "Ex-smoker")
Hint options: combine both tests with &.
Hint options: use the OR operator.

D Negate conditions

yao %>% 
  filter(!(age < 18 | weight_kg < 30))

yao %>% 
  filter(!(highest_education %in% 
           c("Primary", "Secondary")))
32. “Wrapping a condition in !() negates it.”
Hint options: negate the %in% expression.

E Missing values

yao %>% filter(is.na(is_smoker))
yao %>% filter(!is.na(is_pregnant))
34. “Use is.na() rather than == NA to identify missing values.”
Hint options: use is.na(is_smoker).
Hint options: keep values >= 20 OR missing values.
