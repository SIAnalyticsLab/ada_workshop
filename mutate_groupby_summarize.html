dplyr Cheat Sheet: Mutating Columns

Creating, modifying, and recoding variables with confidence

SI ANALYTICS
PAGE 01 / 03
Purpose. Use dplyr::mutate() to create new variables, modify existing variables, and prepare analysis-ready columns.
Core pattern. df %>% mutate(new_column = expression)

A Create and modify numeric variables

yao_height %>% 
  mutate(height_meters = height_cm / 100)

yao_height %>% 
  mutate(height_cm = height_cm + 5)
2. “Using the same name inside mutate() modifies an existing variable.”

B Create Boolean variables

yao %>% 
  mutate(is_child = age <= 18)

yao %>% 
  mutate(has_no_symptoms = symptoms == "No symptoms")

yao %>% 
  mutate(has_any_symptoms = symptoms != "No symptoms")
6. “A Boolean variable contains TRUE and FALSE values.”

C Numeric formulas

yao %>%
  mutate(height_meters = height_cm / 100) %>%
  mutate(bmi = weight_kg / (height_meters)^2)
10. “mutate() can create variables from formulas using existing columns.”

D Convert variable types

yao_bmi %>% 
  mutate(bmi_integer = as.integer(bmi),
         bmi_rounded = round(bmi))
12. “as.integer() truncates decimal values rather than rounding them.”

dplyr Cheat Sheet: Summarising Data

Extracting concise statistics and grouped insights

SI ANALYTICS
PAGE 02 / 03
Purpose. Use summarize() to reduce a dataset to meaningful statistics. Add group_by() when you need one statistic per group.
Core pattern. df %>% summarize(new_name = summary_function(column))

A Summary statistics

yao %>% 
  summarize(mean_age = mean(age))

yao %>% 
  summarize(mean_age = mean(age),
            median_age = median(age))
15. “A summary statistic is a single value that describes a sequence of values.”
Hint options: use mean(), median(), and sd().
Hint options: use min() and max().

B Grouped summaries

yao %>% 
  group_by(sex) %>% 
  summarize(mean_age = mean(age))
19. “Grouping by sex and then summarising returns one statistic for each sex group.”
Hint options: group first, then summarise.
Hint options: group by sex, then use min() and max().

C Missing values in summaries

yao %>% 
  group_by(sex) %>% 
  summarize(total_days_missed = 
              sum(n_days_miss_work, na.rm = TRUE))
22. “Without na.rm = TRUE, some summary functions can return NA when missing values are present.”
Hint options: include na.rm = TRUE.

D Group by more than one variable

yao %>% 
  group_by(sex, neighborhood) %>% 
  summarize(mean_age = mean(age))
24. “You can group by multiple variables by listing them inside group_by().”
Hint options: place both grouping variables inside group_by().

E Summary-statistic reference

FunctionUse
mean()Average
median()Middle value
sd()Standard deviation
min(), max()Range limits
sum(..., na.rm = TRUE)Total while removing missing values

dplyr Cheat Sheet: Counting and Ungrouping

Producing reliable grouped outputs for analysis and reporting

SI ANALYTICS
PAGE 03 / 03
Purpose. Count rows efficiently, count rows meeting conditions, and understand when persistent grouping should be removed.
Key tools. n(), sum(condition), count(), ungroup(), and .drop = FALSE.

A Count rows with n()

yao %>% 
  group_by(neighborhood) %>% 
  summarize(count = n())
27. “n() does not require an argument.”
Hint options: combine n() and mean(..., na.rm = TRUE).

B Count rows meeting a condition

yao %>% 
  group_by(neighborhood) %>% 
  summarize(count_under_18 = sum(age < 18))

yao %>% 
  group_by(neighborhood) %>% 
  summarize(known_treatments = 
              sum(!is.na(treatment_combinations)))
29. Why can sum(age < 18) count children?
Hint options: place the equality condition inside sum().

C Use count() for concise counts

yao %>% count(occupation)
yao %>% count(sex, occupation)
31. “count(occupation) is a concise alternative to group_by(occupation) %>% summarize(n = n()).”

D Remove persistent grouping

yao %>% 
  group_by(sex, neighborhood) %>% 
  summarize(mean_age = mean(age)) %>% 
  ungroup()
34. “After grouped analysis, ungroup() can prevent unexpected downstream behaviour.”

E Include missing combinations

yao_no_male_children %>% 
  mutate(sex = as.factor(sex),
         age_category_3 = as.factor(age_category_3)) %>% 
  group_by(sex, age_category_3, .drop = FALSE) %>% 
  summarize(number_of_individuals = n())
35. “.drop = FALSE can retain factor-level combinations that are absent from the observed rows.”
Hint options: convert the grouping variables to factors, then use .drop = FALSE.
